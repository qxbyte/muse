# Muse 操作手册

> 面向最终用户的"配置 + 命令 + 场景"参考。新装好 muse → 跟着 §1-2 跑通;有具体诉求 → 查 §3-7;不确定怎么用 → §9 找最近的场景。
>
> 文档版本对齐 muse `v0.1.3`。配置覆盖**当前已实装**的字段;后续模块(上下文管理工程 / Agent 记忆系统 / 向量索引 等)落地时会同步追加章节。
>
> 设计文档(架构 / 路线图 / 决策依据)不在本文档范围,见仓库内的设计文档库索引。

## 1. 安装与启动

### 1.1 安装

```bash
# 全局(任意目录都能跑 muse 命令)
npm i -g @qxbyte/muse

# 或仅本仓库使用(项目内 devDependency)
npm i -D @qxbyte/muse
npx muse
```

要求:Node.js >= 20。

### 1.2 首次启动

```bash
muse                          # 进入交互 TUI
muse "总结一下 README"          # 一次性 prompt(类似 cat README | xargs llm)
muse --version                # 打印版本
```

**首次启动报错 `No model configured` / `Active model not found`** — 跳到 §7 配置模型仓库。

### 1.3 交互模式快速键

| 键 | 作用 |
|---|---|
| `Enter` | 提交 |
| `\` + `Enter` | 多行换行 |
| `Esc` | 取消当前 LLM / 工具调用,或清空引导队列 |
| `Esc Esc` | rewind:把最后一条 user message 放回输入框,删除其后所有消息 |
| `Shift+Tab` | 循环切换权限模式(详见 §5) |
| `Ctrl+C` | 退出(或先取消正在跑的工具) |
| `↑` / `↓` | 历史输入翻页 |
| `Cmd+V` / `Ctrl+V` | 粘贴文本;若剪贴板含图自动转 `[Image #N]` 附件 |
| `@<path>` | 文件引用补全:输入 `@src/` 弹出补全 overlay,选中后自动展开文件内容 |
| `/` | Slash 命令补全 |

## 2. 配置体系

### 2.1 5 层叠加(由低到高,高覆盖低)

```
1. 内置默认值                          (代码里 DEFAULT_* 常量)
2. ~/.muse/settings.json               全局
3. <cwd>/.muse/settings.json           项目(可入 git)
4. <cwd>/.muse/settings.local.json     项目本地(chmod 600,自动 gitignore)
5. 环境变量 MUSE_*
6. CLI flags(-m / -p / --mode 等)
```

**不配 = 用默认**。只在想改默认时才写对应字段。绝大多数用户只需 `~/.muse/settings.json` 中配 `llm.model` + 一个 `models.local.json`。

### 2.2 文件位置

| 文件 | 用途 |
|---|---|
| `~/.muse/settings.json` | 跨项目的全局偏好 |
| `~/.muse/settings.local.json` | 全局密钥兜底(chmod 600) |
| `~/.muse/models.local.json` | 模型仓库(详见 §7) |
| `<project>/.muse/settings.json` | 团队共享项目配置(入 git) |
| `<project>/.muse/settings.local.json` | 个人项目配置 + 密钥(不入 git) |
| `~/.muse/projects/<hash>/sessions/*.jsonl` | 会话历史 |
| `~/.muse/projects/<hash>/memory/` | 长期 memory(详见 §8) |
| `~/.muse/logs/<date>.jsonl` | `--debug` 模式日志 |

### 2.3 `${ENV_VAR}` 占位符

settings.json 任意字符串字段可写 `${ENV_VAR_NAME}`,加载时展开。未设置的环境变量保留原样。

```jsonc
{
  "providers": {
    "deepseek": { "apiKey": "${DEEPSEEK_API_KEY}" }
  }
}
```

## 3. Settings 字段大全(已实装)

> 所有字段**可选**;不写 = 默认。下方默认值标在注释里。

### 3.1 `llm` — 主模型配置

```jsonc
{
  "llm": {
    "model": "deepseek-chat",       // 必填一个;对应 models.local.json 的 entry id
    "provider": "deepseek",         // 仅 fallback 路径用(无 models.local.json 时)
    "temperature": 0.7,             // 0~2,默认由 provider 决定
    "maxTokens": 4096               // 输出最大 token,默认由 provider 决定
  }
}
```

### 3.2 `providers` — provider 凭据(仅 fallback 路径用)

主推使用 §7 `models.local.json`;这里仅在没有 registry 时作 fallback。

```jsonc
{
  "providers": {
    "deepseek": {
      "apiKey": "${DEEPSEEK_API_KEY}",
      "baseUrl": "https://api.deepseek.com",
      "extraHeaders": { "X-Custom": "..." }
    }
  }
}
```

### 3.3 `permissions` — 工具权限规则

```jsonc
{
  "permissions": {
    "allow": ["Read", "Grep", "Bash(npm test)"],     // 自动放行
    "ask":   ["Edit", "Write"],                      // 弹确认
    "deny":  ["Bash(rm)", "Bash(sudo:*)"],           // 自动拒绝
    "defaultMode": "ask"                              // ask | strict | relaxed
  }
}
```

**Pattern 格式**:
- `"ToolName"` — 精确工具名(`Read` / `Edit` / 等)
- `"Bash(<prefix>)"` — Bash 工具 + 命令前缀(如 `Bash(git status)`)
- `"Bash(<prefix>:*)"` — 通配后续参数(如 `Bash(npm:*)` 放行所有 npm 命令)

**硬约束**:`rm -rf` / `sudo` / `curl|sh` 等危险命令由 Bash 工具内部 `HARD_DENY_PATTERNS` 拦截,**所有权限模式都绕不过**。

### 3.4 `preprocess` — 4 段消息预处理管道

```jsonc
{
  "preprocess": {
    "input": {
      "atFileExpand": { "enabled": true, "maxBytes": 1048576 },  // @ 文件引用展开
      "templateExpand": { "enabled": true },                      // 模板展开
      "maxChars": 100000,                                         // 输入长度上限
      "redactPreScan": { "enabled": false }                       // 输入侧密钥扫描
    },
    "request": {
      "trimHistory": {
        "enabled": true,
        "budgetRatio": 0.8,                                       // 滑窗裁剪触发阈值
        "targetRatio": 0.6,                                       // 裁到此预算占比以下停手
        "preserveUserMessages": true                              // I-1:被切段的 user 原文保留(false 回旧 marker)
      },
      "budgetGuard": {
        "enabled": true,
        "budgetRatio": 0.95,                                      // 兜底压缩触发阈值
        "promoteFactsToMemory": true                              // I-5:自动 compact 时 promote facts 到 memory
      },
      "compact": {
        "schema": "9-section",                                    // I-2 摘要模板(可选 6-section)
        "fallbackOnFormatFail": true,                             // O3:9-section LLM 失败时降级 6-section 重试
        "dedupPromotedFacts": true                                // O4:同名 memory 已存在 → skip 不覆盖
      },
      "injectMemory": {
        "budgetScaleEnabled": true                                // O1:budget 紧张时按占比动态降量(50%/25%/满)
      },
      "tokenize": {
        "imageTokenEstimate": 1500                                // O2:image part token 估算(默认 1500/张)
      },
      "redact": { "enabled": false },                             // 出口侧密钥扫描(P2)
      "clearStaleToolResults": {                                  // I-3:可重获取的旧 tool result 折叠
        "enabled": true,
        "keepRecentTurns": 3,
        "clearTools": ["Read", "Grep", "Glob"]
      }
    },
    "result": {
      "truncate": { "budgetBytes": 30000 },                       // 单条 tool result 上限
      "detectBinary": { "enabled": true },                        // 二进制检测
      "summarize": { "enabled": true },                           // 自动生成 summary
      "normalizeError": { "enabled": true },                      // 错误归一化
      "redact": { "enabled": true },                              // 结果侧密钥脱敏
      "injectDiff": true                                          // Edit/Write 注入 diff
    },
    "render": {
      "streamMarkdown": { "enabled": true },                      // 流式 markdown 渲染
      "collapseLong": { "enabled": true, "threshold": 200 }       // 长 message 折叠阈值
    },
    "disable": ["redact-pre-scan"]                                // 全局禁用特定 stage(kebab-case)
  }
}
```

### 3.5 `hooks` — 9 个生命周期钩子

```jsonc
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "^Bash",                          // 工具名 regex(可选,默认匹配所有)
        "command": "/path/to/audit.sh",              // shell 命令,stdin 收 JSON ctx
        "timeout": 5000,                             // ms,默认 30000
        "onError": "skip"                            // skip | throw,默认 throw
      }
    ],
    "PostToolUse": [...],
    "PreCompact": [...],            // 可 block compaction
    "PostCompact": [...],           // 不阻断
    "SessionStart": [...],          // 返回字符串注入 system prompt
    "SessionEnd": [...],
    "UserPromptSubmit": [...],
    "PreLLMRequest": [...],
    "PostLLMResponse": [...]
  }
}
```

Hook 子进程**走环境变量白名单**(避免泄露密钥),`settings.env` 注入的变量不会自动透传。

### 3.7 `memory` — 长期记忆配置

muse memory 分**两层独立存储 + 合并召回**:

| scope | 路径 | 范围 |
|---|---|---|
| `project` | `~/.muse/projects/<hash>/memory/` | 当前项目专属 |
| `user` | `~/.muse/memory/` | 跨项目用户级 |

召回时两层合并,**项目优先**(同分时 project 排前)。

**写入入口**:
- LLM 自主 `MemoryWrite`(默认 `scope=project`,个人偏好类强制 `scope=user`)
- 用户 `/remember <text>` 显式触发(默认 `scope=user`,详见 §4.4)
- `/compact` 触发的事实自动 promote 到 memory(`scope=project`)

#### 向量召回:三档 provider 选择

embedding 默认关闭,启用时三选一:

| provider | 体积成本 | 网络需求 | 推荐场景 |
|---|---|---|---|
| **hash-bag** | 0(内置) | 无 | 默认 / 启用 embedding 失败时自动降级到这里 |
| **openai-compatible**(preset)| 0 npm 包(纯 fetch) | 需访问 baseUrl | 真语义召回 — 国内 DashScope/智谱直连,海外 OpenAI,本地 Ollama |
| **local-minilm**(留下批) | ~50MB(需手动 `npm i @huggingface/transformers`)| 无 | 完全离线 + 隐私敏感场景 |

**关键承诺**:不管哪一档,失败都自动降级到全文模式,**永远不阻塞 muse 使用**。

#### 配置字段(`provider="openai-compatible"` + preset 模式)

```jsonc
{
  "memory": {
    "embedding": {
      "enabled": true,
      "preset": "dashscope-v3",          // 选 preset(下面 §9.7 三套清单)
      "apiKey": "${DASHSCOPE_API_KEY}",  // 走环境变量,${VAR} 自动展开
      // 以下都可选 — 不填走 preset 默认
      "dim": 1024,                        // 维度;用户根据模型官方说明覆盖
      "model": "text-embedding-v3",       // 模型名覆盖
      "baseUrl": "https://...",           // baseUrl 覆盖
      "topK": 5,                          // 召回 top-K
      "minMemoryCount": 3,                // < 此值退化到全文
      "maxInjectTokens": 1500             // 注入 token 预算
    }
  }
}
```

**内置 preset**:

| 名字 | 厂商 / 端点 | 默认 dim | 需 key | 备注 |
|---|---|---|---|---|
| `dashscope-v3` | 阿里 DashScope | 1024 | ✅ | 国内直连,中文优化,**便宜** |
| `zhipu-3` | 智谱 BigModel | 2048 | ✅ | 国内直连 |
| `openai-3-small` | OpenAI | 1536 | ✅ | 需梯子,$0.02/1M tokens |
| `openai-3-large` | OpenAI | 3072 | ✅ | 需梯子,质量更高 |
| `ollama-nomic` | Ollama 本地 | 768 | ❌ | 先 `ollama pull nomic-embed-text` |
| `ollama-bge-m3` | Ollama 本地 | 1024 | ❌ | 多语言,先 `ollama pull bge-m3` |
| `local-bge-zh` | `@huggingface/transformers` 本地 | 512 | ❌ | 中文优化;先 `npm i -g @huggingface/transformers` |
| `local-bge-en` | `@huggingface/transformers` 本地 | 384 | ❌ | 英文优化 |
| `local-minilm` | `@huggingface/transformers` 本地 | 384 | ❌ | 通用多语言(小) |
| `local-bge-m3` | `@huggingface/transformers` 本地 | 1024 | ❌ | 多语言强项(较大) |

`dim` 覆盖 preset 默认时,muse HTTP 请求会带 `dimensions` 参数(MRL truncation;仅支持的模型有效)。

#### 行为说明(用户视角)

- 启用 embedding 后,muse 启动时构建 `.index.json` 向量索引落到每层 memory 目录
- 启动时 **probe 校验**:发一个探针请求,验证模型实际维度跟配置一致
- 失败自动降级 + 在 stderr 打印失败原因(见下方 `/memory diagnose`)
- 每轮 LLM 调用前,基于最近用户消息召回最相关的 top-K memory 注入到 system prompt
- 召回内容按 trust 分级注入(trusted 完整 body / verified 摘要 / auto 索引行)
- 总注入超 `maxInjectTokens` 时优先保留 trusted
- memory 数少于 `minMemoryCount` 时自动退化全文注入
- **session 内立即生效**:LLM `MemoryWrite` 或 `/remember` 写入后自动更新索引

#### 启用失败排查 — `/memory diagnose`

启用 embedding 但不工作时,在 muse 里运行:

### 3.8 `skills` — Markdown prompt 包

Skills 是用 markdown 写的工作流程包(`SKILL.md` + YAML frontmatter),与工具系统解耦:
工具是**执行层**,skill 是**流程指令**。LLM 可自动匹配 description 触发,也可由用户
显式 `/skill run <name>` 强制触发。

```jsonc
{
  "skills": {
    "enabled": true,                   // 默认 false;开关全局禁用 → 零开销
    "disabled": ["deploy-prod"],       // 黑名单(personal + project 都跳)
    "personalDir": "~/.muse/skills",   // 跨项目用户级
    "projectDir": ".muse/skills"       // 项目级,相对 cwd
  }
}
```

#### 文件结构

```
~/.muse/skills/deploy-prod/
  SKILL.md          frontmatter + body(必有)
  scripts/          可选辅助脚本,body 内引用
  templates/        可选模板
```

#### SKILL.md 格式(对齐业界开放标准)

```markdown
---
name: deploy-prod
description: Deploy current branch to prod. Use when user asks to ship/release.
allowed-tools: [Bash, Read, Edit]
disable-model-invocation: false
---

# Deploy to Prod

## Steps
1. Run `npm test` — must all pass
2. Run `npm run build`
3. Git tag + push
```

| frontmatter | 用途 |
|---|---|
| `name` | slug,kebab/snake-case 小写;与目录名一致 |
| `description` | LLM 自决依据;10-400 字 |
| `allowed-tools` | 工具白名单;激活期 PermissionGate 只放行此列表(plan 模式取交集) |
| `disable-model-invocation` | true → LLM 看不到,仅 `/skill run <name>` 显式触发 |
| `triggers` | 关键词触发(v0.3.x);本期 LLM 仅按 name 匹配 |

#### 触发方式

1. **LLM 自决**:LLM 在 reasoning 里说出 skill name → 自动激活,body 拼到下一轮 system prompt 尾部
2. **`/skill run <name>`**:用户显式触发(绕过 LLM 文本匹配,`disable-model-invocation: true` 也能触发)

#### 作用域 & 冲突

两层叠加:**project 覆盖 personal**(同名时项目级胜出)。

### 3.9 `mcpServers` — MCP 协议接入

接入第三方 MCP(Model Context Protocol)server,自动把它们暴露的工具注册到 ToolRegistry。
对齐 Claude Desktop / Cursor / Windsurf 配置形态 + muse 自家扩展(`enabled / trust / namespace / timeoutMs`)。

#### 前置:安装 SDK

`@modelcontextprotocol/sdk` 是 **optionalDependency**;`npm i -g @qxbyte/muse` 时**默认会装**。
若需手动单装:`npm i -g @modelcontextprotocol/sdk`。SDK 缺失时 muse 仍能跑(/mcp 提示安装)。

#### 配置字段

```jsonc
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/me/projects"],
      "env": { "DEBUG": "1" },     // 透给 server 子进程
      "enabled": true,              // 默认 true;false 跳过
      "trust": "ask",               // auto | ask(默认) | deny
      "namespace": "fs",            // 可选;工具名变 mcp__fs__*(默认用 key)
      "timeoutMs": 30000            // 单次 tool 调用超时(默认 30s)
    },
    "git": {
      "command": "uvx",
      "args": ["mcp-server-git"]
    }
  }
}
```

#### `trust` 行为对照

| 值 | 行为 |
|---|---|
| `"auto"` | 直接放行(信任 server) — 适合本地 filesystem / git 等无副作用 server |
| `"ask"`(默认) | 走 PermissionPrompt,每次询问用户(同内置工具的默认体验) |
| `"deny"` | 永远拒绝(server 注册了 tool 但全部 reject — 适合临时禁用) |

**注意**:plan 模式下,所有 mcp tool 仍按 `permission` 推断结果过滤(read 类放行,write/execute 拒);
trust=auto 不会让 plan 模式失守。

#### 工具命名空间

server 暴露的每个 tool 自动加前缀:**`mcp__<namespace>__<tool>`**(双下划线,业界共识)。
LLM 看到的工具名形如 `mcp__filesystem__read_file`;两个 server 同名 tool 互不冲突。

#### 懒加载与崩溃自愈

- **启动期**:muse **不**spawn 子进程,仅记 manifest(避免冷启变慢)
- **首次工具调用**:spawn → handshake(initialize)→ listTools → 注册到 ToolRegistry
- **超时 / 子进程崩**:自动反注册 tool + 标 disconnected;下次调用重连一次
- **/mcp disconnect <name>**:用户显式断开(反注册其 tool)

#### 行为说明(用户视角)

```
> 配置 1 个 MCP server,启动 muse
[mcp] 1 server(s) configured (lazy — connect on first tool call)

> LLM 调 mcp__fs__read_file 第一次
(spawn + handshake;首次稍慢)
→ tool result 返回

> /mcp
MCP servers (1):
  ● filesystem
      connected:  true
      tools:      14
      command:    npx -y @modelcontextprotocol/server-filesystem /Users/me/projects
      trust:      ask
```

| `/mcp` | `/mcp <sub>` | MCP servers 管理(详见 §4.6) |
| `/memory` | `/memory <sub>` | 长期记忆管理(详见 §4.3) |
| `/remember` | `<text>` | 让 LLM 把口语化文本结构化后存入 memory(默认 scope=user) |
| `/skill` | `/skill <sub>` | Skills 管理(详见 §4.5) |
| `/resume` | — | 列出可恢复的历史 session,选一个继续 |
| `/mode <mode>` | — | 切权限模式(default / acceptEdits / plan / bypassPermissions) |
| `/btw` | — | 旁白问答 — 临时浮层,不入会话历史 |

### 4.1 `/compact` 详解

```
/compact                       # 默认保留最近 4 条 + LLM 摘要前面
/compact --keep 8              # 保留最近 8 条
```

**触发条件**:
- 自动:`budgetGuard` 阈值(默认 0.95)超出时
- 手动:本命令

**hooks**:`PreCompact`(可 block) / `PostCompact`(不阻断)

### 4.3 `/memory` 子命令

```
/memory                                # 默认 = list(合并两层)
/memory list [--scope project|user|all]  # 列 memory(默认 all)
/memory view <name> [--scope <s>]      # 显示 frontmatter + body(重名时需 --scope 消歧)
/memory edit <name> [--scope <s>]      # 在 $EDITOR 中打开;保存后 auto → verified 自动升级
/memory delete <name> [--scope <s>]    # 删除
/memory promote <name>                 # auto → verified(trust 升级)
/memory promote-scope <name>           # project → user(scope 升级,跨项目生效)
/memory trust <name> <level>           # 显式改 trust(verified | auto)
/memory search <query>                 # 向量索引召回 top-K(显示 scope tag)
/memory diff [--scope <s>]             # 按 updated_at 降序列出(最近改的在前)+ "X 时间前" 标签
/memory diagnose                       # 显示 embedding 状态 + 配置 + 修复建议
/memory help                           # 帮助
```

**`/memory edit` 说明**:用 `$VISUAL` → `$EDITOR` → `vi` 打开 memory 文件。Ink TUI 临时让出 TTY 给编辑器,退出后恢复。保存后 muse 检测文件变化:
- 内容变了 + 原 trust 是 `auto` → 自动升级到 `verified`(source 改 `user-edit`)
- 内容没变 → 无操作
- 编辑器异常退出 → 显示错误信息

**`/memory search` 说明**:基于 EmbeddingProvider 接口的余弦相似度召回。默认 `hash-bag`(本地零依赖,按关键词匹配)。启用真 embedding(DashScope / OpenAI / Ollama 等,见 §3.7 + §9.7)后走向量召回。trust × scope 双重加权,同分时项目 + 高 trust 排前。

**`/memory diagnose` 说明**:启用 embedding 失败时的排查入口。显示当前配置、索引文件状态、live probe 结果、修复建议。

**trust 三级**:
- `trusted` — 来自 `MUSE.md` / `AGENTS.md` / managed config;视为硬约束
- `verified` — 用户编辑 / 显式 promote;视为确认知识
- `auto` — LLM 自动写入,未审核;视为提示线索,新证据可覆盖

**trust 升降规则**:只升不降。一旦升到 `verified`,无法用 `/memory trust ... auto` 降回。`trusted` 是 hierarchy 文件专属(MUSE.md / AGENTS.md),slash 命令不能设置。

**未实装(留下批)**:`edit`(外部编辑器)/ `diff`(git 集成)。

### 4.4 `/remember` — 用户显式记忆入口

```
/remember [--user|--project] <text>
```

**默认 `scope=user`**(用户主动 `/remember` 通常是跨项目偏好)。muse 调当前 LLM 把口语化文本结构化为 `{ name, type, description, body }`,写入 `trust=verified` + `source=user-remember`。启用 embedding 时自动 upsert,**当前 session 立即可召回**。

**示例**:

```
> /remember I prefer pnpm over npm in all projects
Created memory "prefer-pnpm-over-npm"
  scope:       user
  type:        user
  trust:       verified (source: user-remember)
  description: User prefers pnpm package manager over npm across projects.
  → ~/.muse/memory/prefer-pnpm-over-npm.md

> /remember --project this team uses semantic-release for versioning
Created memory "team-uses-semantic-release"
  scope:       project
  ...
```

**与 `MemoryWrite`(LLM 自主)的区别**:`/remember` 是用户主动,自动 `trust=verified`;`MemoryWrite` 由 LLM 在对话中判断写入,默认 `trust=auto`(用户可后续 `/memory promote <name>` 升级)。

### 4.5 `/skill` 子命令

```

```

**示例**:


## 5. 权限模式(4 档)

`Shift+Tab` 循环切换;启动时 `--mode <mode>` 指定;`/mode <mode>` 命令切换。

| 模式 | 行为 | 适用场景 |
|---|---|---|
| `default` | 完全走 `settings.permissions` 规则 | 日常 |
| `acceptEdits` | `Edit` / `Write` 自动 allow,其他走 default | 信任改动,要快 |
| `plan` | **只允许 read 类工具**(Read / Grep / Glob / MemoryRead / WebFetch),写类全 deny | 分析 / 调研 / 让 LLM 给方案 |
| `bypassPermissions` | 除显式 deny 与 Bash 硬拒绝外**全 allow** | 完全信任 / 沙盒环境 |

**注意**:
- 模式**不持久化** — 关 muse 重开回 `default`
- `--mode bypassPermissions` 启动相当于"无管控",仅在你确定的环境用
- 三态规则(allow / ask / deny)+ 4 档模式正交叠加

## 8. Memory(当前版本)

### 8.1 基础概念

| 类型 | 用途 |
|---|---|
| `user` | 用户角色 / 偏好 |
| `feedback` | 验证过的方法 / 反模式 |
| `project` | 项目事实 / 决策 |
| `reference` | 外部系统指针(URL / 路径) |

LLM 自主调 `MemoryWrite` 写入;`MEMORY.md` 索引前 200 行自动注入 system prompt。

### 8.2 文件位置

```
~/.muse/projects/<sha256(cwd)[:16]>/memory/
  MEMORY.md             索引
  user_*.md             user 类
  feedback_*.md         feedback 类
  project_*.md          project 类
  reference_*.md        reference 类
```

每个 .md 带 frontmatter:

```yaml
---
name: user-prefers-tabs
description: 用户偏好制表符缩进
metadata:
  type: feedback
---

(body)
```

### 8.3 手动编辑

直接打开文件改即可,muse 下次启动自动读取。**不要手改 `MEMORY.md` 索引** — 用 `MemoryWrite` 写新条目时索引会自动更新,手改可能在下一轮被覆盖。

### 8.4 后续(v0.2.x)

`MUSE.md` + `AGENTS.md` 5 层 hierarchy / `/memory` slash / trust 分级 / 可选向量索引 — 见设计文档库,实装上线后追加章节。

```bash
muse --quiet "列出 src/ 下所有 TS 文件的导出名" > exports.txt
```

`--quiet` 跳过 banner + 多余装饰,适合管道。

## 10. 故障排查 / FAQ

> 本章目前仅收录已知高频问题,后续按用户反馈补充。

### 10.1 `No model configured` / `Active model "X" not found`

`~/.muse/models.local.json` 缺失或 `settings.llm.model` 跟 registry id 对不上。三选一:
1. 编辑 `~/.muse/settings.json`:把 `llm.model` 改成 registry 已有 id
2. 编辑 `~/.muse/models.local.json`:把某 entry id 改成 settings 中的值
3. 用 `muse --model <id>` 临时覆盖

### 10.2 token 用爆 / 模型报 context exceeded

- 检查 `models.local.json` 里该 entry 的 `contextWindow` 是否设对了
- 改严 `preprocess.request.trimHistory.budgetRatio`(默认 0.8,可降到 0.7)
- 改严 `preprocess.result.truncate.budgetBytes`(默认 30000,可降到 20000)
- 手动 `/compact --keep 4`

### 10.3 修改 settings.json 不生效

- 看是不是改错了文件(`~/.muse/` vs `<cwd>/.muse/`)
- `/config show` 看实际生效来源
- `/config reload` 不重启 muse 重读

### 10.4 启动卡住 / 闪屏 / 渲染异常

- 加 `--debug` 跑一次,看 `~/.muse/logs/<date>.jsonl`
- 临时关 stream-markdown(§9.7)看是否渲染层问题
- 设 `MUSE_DISABLE_CURSOR_BLINK=1` 关光标闪烁

### 10.5 API key 泄露 / 想改

- key 仅走 `~/.muse/models.local.json`(明文或 `${ENV_VAR}`)
- 改完无需重启:`/config reload`
- 日志默认脱敏(前 4 后 4,中间星号)

## 11. 完整 settings.json 示例


### 11.2 团队项目模板(`.muse/settings.json`,入 git)

```jsonc
{
  "permissions": {
    "deny": [
      "Bash(rm:*)",
      "Bash(git push:*)",
      "Bash(npm publish:*)"
    ],
    "ask": ["Edit", "Write"]
  },
  "preprocess": {
    "result": {
      "truncate": { "budgetBytes": 25000 }
    }
  },
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "^(Edit|Write)$",
        "command": "./.muse/hooks/audit-edit.sh",
        "timeout": 5000,
        "onError": "skip"
      }
    ]
  }
}
```
